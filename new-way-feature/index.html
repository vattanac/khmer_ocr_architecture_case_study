The New Way — 2D Feature Maps Preserved as Patches

Instead of flattening the 2D feature map into a single long sequence, the researchers keep
its 2D structure and slice it into vertical patches — one per column group.

🧱 Part 1 — What the CNN produces

The feature map is a 3D grid: Rows × Columns × Channels

After the CNN processes the Khmer image, it outputs a feature map — a 3-dimensional tensor. Our example uses:

Input image:   1 × 32 × 500   (grayscale × height × width)
After CNN:    512 ×  8 ×  64   (channels × rows × columns)
ROWS = 8

The vertical axis. Each row corresponds to a horizontal band of the original image. With 8 rows, the CNN has compressed the original 32 pixel height by 4×.
This is the crucial axis — it encodes WHERE vertically each Khmer component sits.

COLUMNS = 64

The horizontal axis — position along the text line. 64 columns because the CNN compressed the original 500 px width by ~8×. Each column ≈ one "slice" of the text line.

CHANNELS = 512

At every (row, column) position there are 512 numbers — features learned by the CNN describing edges, curves, strokes etc. at that location. Think of it as 512 "lenses" all looking at the same spot.

feature map 8 × 64 × 512 enters next step
⚠️ Part 2 — Quick Recap: Why the Old Way Failed

Old way: collapse 8 rows → 1 row, then flatten to 1D

Old PATCH_ENC:   8 × 64 × 256  →  248 × 256  (rows collapsed, n becomes large)

Collapsing the 8 rows into 1 by averaging or concatenating destroys vertical position information. The model can no longer tell whether a feature came from the top of the image (diacritics) or the bottom (subscripts). For Khmer, this matters enormously.

After collapsing, a feature that was at row 1 (diacritic zone) and a feature at row 7 (subscript zone) look identical to the Transformer — they are just entries in the same long 1D vector. The vertical structure is lost forever.
the new approach preserves all 8 rows
✂️ Part 3 — The Key Idea: Slice into Vertical Patches

Each patch = ALL 8 rows × a few columns × 512 channels

Instead of collapsing rows, the researchers slice the feature map into vertical strips (patches). Each patch spans all 8 rows but only a small number of columns. With patch width = 1 column, we get exactly 64 patches.

STEP-BY-STEP PATCH CREATION:
1
Take feature map F of shape 8 rows × 64 cols × 512 ch
2
Slice along columns: patch c = F[ all 8 rows, column c, all 512 channels ] → shape [8 × 1 × 512]
3
Flatten each patch: 8 × 1 × 512  →  4096-dimensional vector  (one vector per column)
4
Stack all 64 vectors → sequence of shape 64 × 4096. This is the input to the Transformer.  n = 64 (much smaller than old n = 248!)
64 patches × 4096-dim vectors enter Transformer
📐 Part 4 — What Each Row Encodes: Vertical Khmer Structure

The 8 rows map to distinct Khmer character zones

This is the key insight of the paper. Khmer characters stack vertically: diacritics above, base consonant in the middle, subscripts below. The 8 rows of the feature map directly correspond to these zones. A patch vector of size 4096 = 8 groups × 512 — each group encodes one row's features.

Row 1 Top of image (px 0–4) Upper diacritics, tone marks ៉ ៊ ់ ័ vector[0 … 511]
Row 2 Upper-mid (px 4–8) Vowels above consonant ិ ី ឹ ឺ vector[512 … 1023]
Row 3 Upper body (px 8–12) Top of base consonants, vowel ៀ ើ ក ខ ស ៀ vector[1024 … 1535]
Row 4 Mid body (px 12–16) Main body of base consonants (1st + 2nd series) ក គ ប ស vector[1536 … 2047]
Row 5 Lower body (px 16–20) Lower strokes + vowels ា ុ ូ ង ណ ា ុ vector[2048 … 2559]
Row 6 Upper subscript (px 20–24) Coeng (្) connector + top of subscript ្ក ្គ ្ត ្ន vector[2560 … 3071]
Row 7 Lower subscript (px 24–28) Main body of subscript consonants (32 forms) ្រ ្ល ្វ ្ម vector[3072 … 3583]
Row 8 Bottom (px 28–32) Below-vowels ួ ូ, Khmer numerals ០–៩ ុ ូ ួ ០ vector[3584 … 4095]
Why this matters for Khmer: When the Transformer receives a patch vector of 4096 numbers, the numbers at positions 1536–2047 (Row 4) encode the base consonant body, while the numbers at 3072–3583 (Row 7) encode any subscript. These are different dimensions of the same vector — the Transformer's self-attention can learn to attend to both simultaneously when decoding a consonant cluster.

⚖️ Part 6 — Old Way vs New Way: Side by Side

The complete comparison

❌ Old Way (1D collapse)

  • Feature map 8×64×256
  • 8 rows collapsed → averaged into 1
  • Sequence: 248 × 256
  • Vertical structure destroyed
  • ក and ្រ share the same dims → confused
  • Self-attention cost: 248² = 61,504
  • Khmer clusters often misread

✅ New Way (2D patches)

  • Feature map 8×64×512
  • 8 rows preserved in each patch
  • Sequence: 64 × 4096
  • Vertical structure preserved
  • ក at dims 1536–2047, ្រ at 3072–3583 → separate
  • Self-attention cost: 64² = 4,096 (15× less!)
  • Khmer clusters decoded correctly

🔑 Summary: Why 2D Patches are the Right Approach for Khmer
1. The CNN produces a 3D feature map (Rows × Cols × Channels) that preserves vertical spatial structure.
2. Instead of collapsing rows, the map is sliced into vertical patch vectors — each patch spans ALL rows.
3. Each patch vector's dimensions are partitioned by row: base consonant features and subscript features occupy different, non-overlapping ranges.
4. The Transformer's self-attention can now directly compare the base consonant zone and subscript zone within a single patch, correctly reading clusters like ក្រ, ស្ត, ប្រ.
5. Bonus: n=64 patches instead of n=248 → self-attention cost drops from 61,504 to 4,096 pairs — a 15× reduction.
