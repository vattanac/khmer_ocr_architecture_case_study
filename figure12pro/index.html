How AI Reads & Understands
Khmer Text from Images

This architecture is like a very smart reader — it looks at a picture of Khmer script and outputs the actual characters, step by step. Example: ប៉ុស្តិ៍សង្កាត់បឹងកក់២ (Sangkat Boeng Kak 2 Post Office)

ប៉ុស្តិ៍សង្កាត់បឹងកក់២ → "ប ៉ ុ ស ្ត ិ ៍ ស ង ្ក ា ..."
🧑‍🏫

The Big Analogy — Think of a Smart Student Reading Handwriting

Imagine you show a student a handwritten note in a language they know.
1️⃣ Their eyes scan the page and spot the overall shapes. (CNN)
2️⃣ They split it into small regions, like reading word by word. (PATCHENC)
3️⃣ Their brain connects the regions to understand context ("this curve + that line = ណ"). (TRENC)
4️⃣ They write out each character one by one, checking what they've written so far. (TRDEC)

Step-by-Step Deep Dive
1
CNN Feature Extractor — "The Eyes"
🖼 CNN Feature Extractor — Architecture Diagram

The image is fed into a Convolutional Neural Network (CNN) — think of this as the AI's eyes. Instead of reading pixels one by one, the CNN scans the whole image with sliding filters, detecting edges, curves, and strokes. It outputs a compressed feature map F — a grid where each cell encodes "what's interesting at this location."

📐 If the input image is 32×128 pixels, the CNN shrinks it to about 8×32 feature cells (downsampled 4×), but each cell now holds a rich 512-dimensional description instead of just 1 pixel value.

2
PATCHENC Patching Module — "Slice the Image into Columns"
🖼 PATCHENC — Patching Module Diagram

The feature map is 2D (like a table). But Transformers need a 1D sequence (like a list of words). The Patching Module slices the 2D feature map into narrow vertical strips (patches), then flattens each strip into a single vector.

🖼 Example: If the feature map is 8 rows × 32 cols, and patch size is (4, 1), each patch = 4 rows × 1 col. After flattening + a linear layer, each patch becomes a vector Pi in ℝD. You end up with 32 patches in a sequence — like 32 "tokens" in a sentence.

3
TRENC Transformer Encoder — "Understanding Context"
🖼 TRENC — Transformer Encoder Diagram

Now we have a sequence of patches P₁…Pₙ. The Transformer Encoder uses self-attention to let every patch "look at" every other patch. This is how the AI understands context: "this stroke here at P5 is connected to the curve at P7, so together they form the character ណ."

🧠 Think of it like a reading comprehension exercise: each word re-reads the entire sentence to understand its own meaning better. The output Ti is a richer, context-aware version of each patch.

4
TRDEC Transformer Decoder — "Writing the Answer"
🖼 TRDEC — Transformer Decoder Diagram

The decoder takes two inputs: the contextual features T (what the image says) and the context sequence Z (characters already written). It predicts the next character one at a time — this is called autoregressive decoding.

✍️ It's like a translator who reads the original text (T) and writes the translation one word at a time, always looking back at what they already wrote (Z) to make sure it's consistent. Z = [SOS] kicks it off, the model writes ប → ៉ → ុ → ស → ្ត… and stops when it writes [EOS].

The Z arrow you see entering the side of TRDEC in the figure represents this "what I've written so far" feedback loop.

5
OUTPUT Predicted Characters — "The Final Answer"

At each decoding step, the model outputs Ŷi ∈ ℝC — a probability distribution over all C = 131 Khmer character classes. The character with the highest probability is picked (this is called argmax).

The model is trained using Cross-Entropy Loss: for each character position, it gets penalized when it predicts the wrong character. Over millions of training examples, it learns to be very accurate.

Common Questions
Why use CNN instead of just the Transformer from the start?
Images have too many raw pixels (e.g. 32×256 = 8,192 pixels). Transformers would be way too slow processing every pixel. The CNN efficiently compresses this into a small 8×64 feature map first, then the Transformer handles it.
What is "D" and why does everything use ℝD?
D is the hidden dimension size — typically 256 or 512 numbers. Every patch, every feature, every token is represented as a list of D numbers. This uniform size lets all layers easily pass data to each other.
Why is the patching (2,1), (3,1), or (4,1) and not square?
Khmer text is written horizontally. Tall narrow patches (k₁ rows × 1 col) capture each character column naturally — a single Khmer character typically spans 2–4 rows of features, but only 1–2 columns.
What is the Z arrow going into TRDEC?
Z is the characters already predicted: Z = [SOS, ប, ូ, ល, …]. The decoder uses this to avoid repeating itself and to maintain grammatical consistency. It's like a human writer rereading their own sentence before writing the next word.
How does the model know when to stop?
The model learns to predict a special [EOS] (End of Sequence) token when it's finished. During training, ground truth text always ends with [EOS], so the model learns this pattern. At inference, decoding stops as soon as [EOS] is predicted.
What are the 131 character classes (C=131)?
These are the 131 possible Khmer script characters and symbols the model can output, plus special tokens like [SOS], [EOS], and possibly space/punctuation. At each step, the model assigns a probability to every one of them.
