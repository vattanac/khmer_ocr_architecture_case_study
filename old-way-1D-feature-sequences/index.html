Khmer OCR — Architecture Deep Dive

The Old Way: 1D Feature Sequences

How the baseline OCR model converts an image into a feature sequence — and why it struggles with wide Khmer text lines.

Buoy et al., "Toward a Low-Resource Non-Latin-Complete Baseline: An Exploration of Khmer OCR," IEEE Access Vol. 11, 2023
🎯 First — What Does "Feature Sequence" Even Mean?

A computer cannot "read" an image directly. The OCR model first needs to translate the image into numbers. Those numbers are called features. When they are arranged in a list (one after another), it becomes a feature sequence.

💡
Simple analogy

Think of the image as a long sentence written on a strip of paper. A feature sequence is what you get when you describe each small section of that strip in numbers. Instead of "ក looks like this", you get "position 5 → [0.8, 0.1, 0.4, …, 0.9]" — a vector of D numbers that encodes what the model saw at that position.

📷 Input — Khmer text line image
តាកែវ គឺ ស៊ីសាមុត ប្រជាពលរដ្ឋ
H = 32 px
W = 500 px
⚠️ Khmer text lines are very wide because there are no spaces between words.
🔄 The Old Way — 5 Steps
1

INPUT — Full-width Khmer text line image

The model receives the complete text line as a single image. In the old way, no splitting or chunking is done. The entire width is processed at once.

Height H = 32 px Width W = 500 px → one image, full width

The wider the text line, the harder the problem — and Khmer lines can be very wide because there are no word spaces.

2

CNN — Extract a 2D feature map

A Convolutional Neural Network (CNN) scans the image like a magnifying glass, picking up edges, curves, and shapes. It outputs a smaller grid of features — not raw pixels, but learned descriptions.

Image (H × W × 1) → Feature Map (H' × W' × C_feat)
Example: 32 × 500 × 1 pixel image → 4 × 62 × 256 feature map
The CNN shrinks the spatial size (÷8 each) but adds depth (C_feat = 256 channels).

Each cell in the 4×62 grid holds 256 numbers (one per channel) describing what the CNN detected in that patch of the image.

3

PATCH_ENC — Flatten the grid into a sequence

The 2D feature map (a grid) must be turned into a 1D sequence (a flat list) so the Transformer can read it. PATCH_ENC does this by reading the grid column by column (or patch by patch) and stacking everything into a sequence of vectors.

Feature Map (4 × 62 × 256) → Sequence (n × D)
n = 4 × 62 = 248 vectors | D = 256 (feature dimension per vector)
Each of the 248 vectors is a D-dimensional number array summarising one image patch.
📊 Feature sequence — n = 248 vectors, each of dimension D = 256
Each column above = one feature vector (position in the sequence). Height = depth D. Value = what the model "saw" at that position.
📌
What is D?

D is the dimension of each feature vector — how many numbers are inside one vector. In this paper D = 256. It is fixed by the model design. Every position in the sequence (from position 1 to n) has exactly D numbers inside it. So "D feature sequence" = a sequence of n vectors, each of length D.

4

TR_ENC — Transformer Encoder with Self-Attention (⚠️ The Problem)

The Transformer Encoder reads the sequence of n = 248 vectors. It uses self-attention to let every vector "look at" every other vector and decide what is important.

🔴 Self-Attention Grid — every vector attends to every other vector
n = 16 (small example)
16 × 16 = 256 pairs ✓

For the full sequence (n = 248), every one of the 248 vectors must compute attention with every other 247 vectors:

Attention cost = n × n = n²
n = 248 → 248² = 61,504 pairs
n = 500 → 500² = 250,000 pairs
n = 1000 → 1000² = 1,000,000 pairs
⚠️
O(n²) Complexity — why it is a problem

If you double the width of the image (e.g., a longer Khmer sentence), the sequence length n doubles, and the self-attention cost quadruples (n² = (2n)² ÷ n² = 4×). Memory and time grow explosively for long Khmer text lines.

5

TR_DEC — Decoder generates text character by character

The Transformer Decoder reads the encoded sequence from TR_ENC and generates the output text one character at a time. At each step, it outputs a probability over all C = 131 Khmer characters.

Encoded sequence (n × D) → Text: "តាកែវ គឺ ស៊ីសាមុត …"
TR_DEC outputs one character per step, then feeds it back as input for the next step (autoregressive).

The decoder itself is not the bottleneck — the bottleneck is the O(n²) self-attention in TR_ENC when n is large.

🔢
The maths of O(n²)

If W = 100 px needs 169 attention pairs, then W = 2000 px (20× wider) needs 62,500 pairs — that is 370× more computation, not 20×. This is what O(n²) means: cost grows with the square of length, not linearly.

🔑 What Exactly is D?

The letter D appears in the paper as the dimension of each feature vector in the sequence. Here is what that means:

Term Meaning Example value
n Number of vectors in the sequence (sequence length). Grows with image width. This is the problem. 248 vectors
D Number of numbers inside each vector (vector dimension). Fixed by model design — does not change with image width. D = 256
Feature sequence The full list: n vectors × D numbers each. Written as a matrix of shape n × D. 248 × 256
Self-attention cost Grows as n² — squaring the sequence length. D does not affect this cost directly. n² = 61,504
So when the paper says "D feature sequences" — it means: a sequence of feature vectors, where each vector has D dimensions. The problem in the old way is not D (that is fixed and small), but n (which grows with image width and makes self-attention expensive).

✅ Summary in Plain Language

In the old way, the OCR model takes the full-width Khmer image and runs it through a CNN to get a feature map. That map is then flattened into a sequence of n vectors, each vector having D = 256 numbers (the "D feature sequence"). The Transformer Encoder then runs self-attention on those n vectors — but self-attention compares every vector to every other, so it costs n² in memory and time. Since Khmer text lines are wide (no spaces between words), n is large, and n² is huge. This is the core problem the chunking approach (the new way, Figure 15) solves — by splitting the image into short chunks first, so n stays small.
