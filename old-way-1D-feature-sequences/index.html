How traditional OCR tried (and struggled) to read Khmer text

🎯 FIRST — What Does "Feature Sequence" Even Mean?

A computer cannot "read" an image directly. The OCR model first needs to translate the image into numbers. Those numbers are called features. When they are arranged in a list (one after another), it becomes a feature sequence.

💡 Simple Analogy

Think of the image as a long sentence written on a strip of paper. A feature sequence is what you get when you describe each small section of that strip in numbers. Instead of "ក looks like this", you get:

📌 What is D?
D = 256 is the dimension of each feature vector — how many numbers are packed inside one position. Every position (1 to n=248) has exactly 256 numbers describing that image patch.
So the full sequence = 248 positions × 256 numbers each.

04
TR_ENC — Transformer Encoder (⚠️ The Bottleneck)
Self-Attention · O(n²) complexity · explodes for long Khmer lines
The Transformer Encoder reads all 248 vectors and uses self-attention to let every vector "look at" every other vector. This is powerful but expensive — it computes n × n pairs.

TR_DEC — Decoder generates text character by character
Autoregressive · one character per step · outputs over 131 Khmer classes
The Transformer Decoder reads the encoded sequence and generates output text one character at a time. At each step it outputs a probability over all C = 131 Khmer characters, then feeds that character back as input for the next step. The decoder itself is not the bottleneck — the bottleneck is TR_ENC's O(n²) attention.

The maths of O(n²)
If W = 100 px needs 169 attention pairs, then W = 2000 px (20× wider) needs 62,500 pairs — that is 370× more computation, not 20×. This is what O(n²) means: cost grows with the square of length, not linearly.
🔑 What Exactly is D ?

The letter D appears in the paper as the dimension of each feature vector in the sequence. Here is what that means:

TermMeaningExample value
n Number of vectors in the sequence (sequence length). Grows with image width. This is the problem. 248 vectors
D Number of numbers inside each vector (vector dimension). Fixed by model design — does not change with image width. D = 256
Feature sequence The full list: n vectors × D numbers each. Written as a matrix of shape n × D. 248 × 256
Self-attention cost Grows as n² — squaring the sequence length. D does not affect this cost directly. n² = 61,504
So when the paper says "D feature sequences" — it means: a sequence of feature vectors, where each vector has D = 256 dimensions. The problem in the old way is not D (that is fixed and small), but n (which grows with image width and makes self-attention expensive).

✅ Summary in Plain Language
In the old way, the OCR model takes the full-width Khmer image and runs it through a CNN to get a feature map. That map is then flattened into a sequence of n vectors, each vector having D = 256 numbers (the "D feature sequence"). The Transformer Encoder then runs self-attention on those n vectors — but self-attention compares every vector to every other, so it costs n² in memory and time. Since Khmer text lines are wide (no spaces between words), n is large, and n² is huge. This is the core problem the chunking approach (the new way, Figure 15) solves — by splitting the image into short chunks first, so n stays small and cost stays linear O(n) instead of quadratic.
